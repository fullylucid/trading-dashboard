================================================================================
API ENDPOINTS
================================================================================

GET /api/signals/
  → Latest signal feed (limit, min_confidence filters)

GET /api/signals/{symbol}
  → Latest signal with full component breakdown

GET /api/signals/{symbol}/history?period=7d
  → Historical signals (1d, 7d, 30d)

GET /api/scanner/{type}?symbol=AAPL
  → Scanner-specific detailed output
  → Types: smart_money, options, sec, sentiment, short_interest, news, technical, quant_ensemble

WebSocket /api/ws/signals
  → Real-time signal streaming
  → Subscribe: {"action": "subscribe", "symbols": ["AAPL", "TSLA"]}

POST /api/telegram/webhook
  → Incoming Telegram commands

SIGNAL MODEL

Each signal includes:

  "id": "uuid",
  "timestamp": "2026-05-24T14:32:15Z",
  "symbol": "AAPL",
  "signal": "buy|sell|hold",
  "confidence": 72,  <- 0-100 percent
  "price": 150.23,
  "reason": "Quant ensemble + smart money accumulation",
  "components": {
    "quant_ensemble": {confidence, momentum, pattern, regime, ...},
    "smart_money": {insider_buys, concentration, volume_ratio, ...},
    "options": {unusual_volume, put_call_ratio, skew, ...},
    ... (each scanner breakdown)
  },
  "alerts_sent": ["telegram"]
}
